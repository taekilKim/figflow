# 🌐 웹에서 바로 사용하기

npm 설치 없이 바로 웹 브라우저에서 FigFlow를 사용할 수 있습니다!

## 🚀 바로 사용하기

**웹 주소:** https://taekilKim.github.io/figflow/

위 링크를 클릭하면 바로 FigFlow를 사용할 수 있습니다.
설치나 설정이 전혀 필요 없습니다!

## ✨ 웹 버전에서 할 수 있는 것

### 1. 즉시 체험 가능
- ✅ 3개의 샘플 노드로 플로우차트 편집
- ✅ 노드 드래그, 연결선 그리기
- ✅ 줌/팬으로 캔버스 조작
- ✅ 속성 편집 (제목, 상태, 메모)
- ✅ 프로젝트 저장 (브라우저에 자동 저장됨)

### 2. Figma 연동 (선택사항)
Figma에서 실제 프레임 썸네일을 가져오려면:

1. **Figma Access Token 발급**
   - https://www.figma.com/developers/api#access-tokens 접속
   - "Generate new token" 클릭
   - 토큰 복사

2. **FigFlow에서 Sync**
   - 웹 페이지 상단의 "Sync" 버튼 클릭
   - 복사한 토큰 입력
   - 썸네일 자동 업데이트!

## 💾 데이터는 어디에 저장되나요?

모든 데이터는 **브라우저의 localStorage**에 저장됩니다:
- ✅ 인터넷이 끊겨도 작업 가능
- ✅ 서버에 데이터를 보내지 않음 (100% 로컬)
- ✅ 같은 브라우저에서 다시 접속하면 저장된 내용 그대로 유지

⚠️ **주의사항:**
- 브라우저 캐시를 삭제하면 데이터가 사라집니다
- 다른 브라우저나 시크릿 모드에서는 데이터가 공유되지 않습니다

## 🔄 자동 배포

이 웹 페이지는 GitHub Actions를 통해 자동으로 배포됩니다:
- 코드가 업데이트될 때마다 자동으로 새 버전이 배포됩니다
- 브라우저를 새로고침하면 최신 버전을 사용할 수 있습니다

## 🛠️ 개발자용: 로컬에서 실행하기

npm이 설치되어 있다면 로컬에서 실행할 수도 있습니다:

```bash
# 1. 의존성 설치
npm install

# 2. 개발 서버 실행
npm run dev

# 3. 브라우저에서 열기
# http://localhost:5173
```

## 📱 모바일에서도 사용 가능

웹 버전은 모바일 브라우저에서도 접속 가능합니다!
- 스마트폰이나 태블릿에서도 https://taekilKim.github.io/figflow/ 접속
- 터치로 노드 드래그, 핀치 줌 지원

## ❓ 문제 해결

### 페이지가 로딩되지 않을 때
- 브라우저를 새로고침 (Ctrl+F5 또는 Cmd+Shift+R)
- 다른 브라우저에서 시도 (Chrome, Firefox, Safari 등)

### 저장한 데이터가 사라졌을 때
- 브라우저 캐시가 삭제되었을 가능성이 있습니다
- 중요한 작업은 "Export" 기능으로 백업하세요 (향후 추가 예정)

### Sync가 작동하지 않을 때
- Figma Access Token이 올바른지 확인
- 브라우저 콘솔(F12)에서 에러 메시지 확인
